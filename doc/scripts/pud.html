Intro - Introduction to function for pud converting

This contains everything about converting from wc2.

Functions

DefineAiWcNames( "ai-0", "ai-1", ... )

Define AI mapping from original number to internal symbol. The parameters are simply a list of ai names. The first name coresponds to pud ai number 0, the second name coresponds to pud ai number 1, and so on.

Example

    DefineAiWcNames(
     "land-attack", "passive", "orc-03", "hum-04", "orc-04", "hum-05", "orc-05",
     "hum-06", "orc-06", "hum-07", "orc-07", "hum-08", "orc-08", "hum-09", "orc-09",
     "hum-10", "orc-10", "hum-11", "orc-11", "hum-12", "orc-12", "hum-13", "orc-13",
     "hum-14-orange", "orc-14-blue", "sea-attack", "air-attack", "hum-14-red",
     "hum-14-white", "hum-14-black", "orc-14-green", "orc-14-white", "orc-exp-04",
     "orc-exp-05", "orc-exp-07a", "orc-exp-09", "orc-exp-10", "orc-exp-12", "orc-exp-06a",
     "orc-exp-06b", "orc-exp-11a", "orc-exp-11b", "hum-exp-02a-red", "hum-exp-02b-black")

DefineConstructionWcNames( "construction-0", "construction-1", ... )

Define construction mapping from original number to internal symbol. The parameters are simply a list of construction names. The first name coresponds to wc2 construction 0, the second name coresponds to construction 1, and so on.

Example

    DefineConstructionWcNames("human-shipyard", "orc-shipyard")

DefineIconWcNames("icon-0", "icon-1", ... )

Define icon mapping from original number to internal symbol. The parameters are simply a list of icon names. The first name coresponds to wc2 icon 0, the second name coresponds to icon 1, and so on.

NOTE:In the original there are only 195 icons

Example

    ;; This is the default icon mapping.
    DefineIconWcNames("icon-peasant", "icon-peon", "icon-footman")

DefineMissileTypeWcNames("missile1", "missile2", ... )

Define missiletype mapping from original number to internal symbol. The parameters are simply a list of missile names. The first name coresponds to wc2 missile 0, the second name coresponds to missile 1, and so on.

Example

    DefineMissileTypeWcNames("missile-lightning", "missile-griffon-hammer")

DefineTilesetWcNames("tileset-0", "tileset-1", ... )

Define tileset mapping from original number to internal symbol. This is FIXED to 4 tileset, bad.
tileset-0
Name of the tileset assigned to index 0 (summer).
tileset-1
Name of the tileset assigned to index 1 (winter).
tileset-2
Name of the tileset assigned to index 2 (wasteland).
tileset-3
Name of the tileset assigned to index 3 (orc swamp).

Example

    -- This is the default tileset mapping.
    DefineTilesetWcNames("tileset-summer", "tileset-winter", "tileset-wasteland", "tileset-swamp")

DefineUpgradeWcNames("upgrade-0", "upgrade-1", ... )

Define upgrade mapping from original number to internal symbol. The parameters are simply a list of upgrade names. The first name coresponds to wc2 upgrade 0, the second name coresponds to upgrade 1, and so on.

Example

    DefineUpgradeWcNames("upgrade-sword1", "upgrade-sword2")

DefineUnitTypeWcNames("unit-0", "unit-1", ... )

Define unit-type mapping from original number to internal symbol. The parameters are simply a list of upgrade names. The first name coresponds to wc2 unit 0, the second name coresponds to wc2 unit 1, and so on.

Example

    DefineUnitTypeWcNames("unit-footman", "unit-grunt")
