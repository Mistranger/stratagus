add-objective

Description

Add an objective to the scenario objectives text list.

Syntax

(add-objective objective position)
objective
Objective text.
position
Optional position (0 offset) to add the objective, default is at the end.

Example

    (add-objective "-Build a barracks")
    (add-objective "-Destroy all enemies" 0)

Adds the objective text "-Build a barracks" to the end of the objectives list and adds the objective "-Destroy all enemies" to the first position.

Not Used

define-construction

Description

Defines a construction.

Constructions can specify multiple files and shadow files for use with different tilesets. Note that shadows are optional.

The construction frames should be ordered by increasing percentages and the first frame should be 0.

Syntax

(define-construction ident
  'file '(
    tileset tileset
    file filename
    size (x y))
  'shadow-file '(
    tileset tileset
    file filename
    size (x y))
  'constructions (list
    '(percent percent
      file construction-file
      frame frame-number))
ident
Unique name of the construction.
tileset
Name of tileset or default to use with any tileset.
filename
Path of the graphic.
percent
Percentage of construction complete.
construction-file
Can be construction or main, specifies which image should be displayed.
frame-number
Specfies which frame number to display.

Example

    (define-construction 'construction-land
      'file '(
        tileset summer
        file "neutral/buildings/land construction site.png"
        size (64 64))
      'file '(
        tileset winter
        file "tilesets/winter/neutral/buildings/land construction site.png"
        size (64 64))
      'constructions (list
        '(percent 0
          file construction
          frame 0)
        '(percent 25
          file construction
          frame 1)
        '(percent 50
          file main
          frame 1)))

Defines construction-land with different graphics for summer and winter tilesets. Buildings 0-24 percent complete use frame 0 of the construction image, 25-49 percent complete use frame 1 of the construction image, and 50-99 percent complete use the frame 1 of the main building graphic.

define-race-names

Description

Define the race names.

Syntax

(define-race-names
  'race '(
    race race-number
    name race-name
    display race-display-name
    [visible])
  ['race '(...)] ...)
race-number
Internal race number. Used to map pud race numbers.
race-name
Internal race name used by other ccl functions.
race-display-name
Name of the race that gets displayed to the user.
visible
If specified the race will be visible in the race menus.

Example

    (define-race-names
      'race '(
        race 0
        name "alliance"
        display "Alliance"
        visible)
      'race '(
        race 1
        name "mythical"
        display "Mythical"
        visible)
      'race '(
        race 2
        name "neutral"
        display "Neutral"))

Defines the alliance, mythical, and neutral races. The neutral race is not visible in the race menus.

Used

$LIBARYPATH/ccl/wc2.ccl

define-ranks

Description

Define ranks for a race.

Syntax

(define-ranks race '(score rank [score rank] ...))
race
The race to define the ranks for.
score
If the player's score is greater than or equal to this number then the rank is displayed. Scores are expected to be sorted in increasing order.
rank
The rank that gets displayed.

Example

    (define-ranks 'alliance '(
    0 Pathetic
    3000 "Not Bad"
    10000 Great
    ))

Defines the ranks for the alliance race.
